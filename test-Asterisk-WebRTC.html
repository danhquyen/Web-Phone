Asterisk WebRTC Softphone (JsSIP, direct — no Siperb)

Status: Idle
Connection
Dialpad

When idle, the keys build the number in Destination. During a call, they send DTMF tones (use these to navigate the PBX auto-attendant / IVR menus).

Log
Required Asterisk server-side configuration (pjsip.conf / http.conf) 
; --- http.conf : enable the WebSocket (WSS) listener ---
[general]
enabled=yes
bindaddr=0.0.0.0
tlsenable=yes
tlsbindaddr=0.0.0.0:8089
tlscertfile=/etc/asterisk/keys/asterisk.crt
tlsprivatekey=/etc/asterisk/keys/asterisk.key
; WebSocket path is /ws  ->  wss://your-host:8089/ws

; --- pjsip.conf : WSS transport ---
[transport-wss]
type=transport
protocol=wss
bind=0.0.0.0

; --- pjsip.conf : a WebRTC endpoint (extension 1001) ---
[1001]
type=endpoint
context=from-internal
disallow=all
allow=opus,ulaw
auth=1001
aors=1001
webrtc=yes            ; enables DTLS-SRTP, ICE, rtcp-mux, AVPF automatically

[1001]
type=auth
auth_type=userpass
username=1001
password=secret

[1001]
type=aor
max_contacts=1
remove_existing=yes

; Notes
; - webrtc=yes implies: media_encryption=dtls, dtls_auto_generate_cert=yes,
;   ice_support=yes, use_avpf=yes, rtcp_mux=yes, media_use_received_transport=yes.
; - The browser requires WSS (TLS). A self-signed cert must be trusted by the
;   browser first, or use a real cert (Let's Encrypt).
; - The page must be served over https:// (or http://localhost) for getUserMedia.
